# Refinex-DBFlow 后台管理原型信息架构

> 阶段：P09.1 后台管理信息架构与页面清单  
> 边界：本目录仅沉淀设计资产，不包含生产代码、后端接口实现或 Thymeleaf 模板。  
> 参考气质：企业内网运维控制台，接近 Nacos 控制台的安静、高密度、可扫描工作台，不做营销页、大 hero 或装饰性首页。

- `docs/exec-plans/specs/2026-04-29-dbflow-mcp-architecture-design.md`
- `docs/ARCHITECTURE.md`
- `0-1-PLANS.md` 的 P09 后台管理 HTML 原型设计阶段

## 设计原则

1. **运维优先**：首屏直接进入可操作信息，不使用品牌展示、营销文案或大面积视觉背景。
2. **密度优先**：表格、筛选器、状态 chip、详情页采用紧凑布局，适合管理员重复查询、批量确认和追踪问题。
3. **安全优先**：Token 明文只允许在签发成功后的单次弹窗中出现；配置页不展示数据库密码、完整连接串或敏感 secret。
4. **审计可追踪**：所有 SQL、安全策略、授权和 Token 操作都能从页面跳转到审计记录，核心标识包括 `request_id`、`sql_hash`、
   `user_id`、`token_id`、`project/env`。
5. **上下文固定**：后台顶部始终展示当前管理员、系统健康摘要和全局搜索入口；涉及数据源的页面必须显式展示 `project/env`。
6. **失败可解释**：错误状态必须展示可执行原因，例如未授权、策略拒绝、配置缺失、数据源不可达、审计查询超时。

## 全局信息架构

### 导航结构

| 分组    | 页面       | 路由建议                        | 说明                        |
|-------|----------|-----------------------------|---------------------------|
| 入口    | 登录       | `/login`                    | 管理员登录与会话过期重登              |
| 工作台   | 总览       | `/admin`                    | 系统状态、风险概览、最近活动            |
| 身份与访问 | 用户管理     | `/admin/users`              | 管理后台用户生命周期                |
| 身份与访问 | Token 管理 | `/admin/tokens`             | 签发、查看、吊销 MCP 访问 Token     |
| 身份与访问 | 项目环境授权   | `/admin/grants`             | 维护用户到 `project/env` 的访问关系 |
| 配置与策略 | 配置查看     | `/admin/config`             | 查看脱敏后的项目、环境、数据源配置         |
| 配置与策略 | 危险策略     | `/admin/policies/dangerous` | 查看高危 SQL 白名单、确认策略和拒绝原因    |
| 审计    | 审计列表     | `/admin/audit`              | 多条件查询审计事件                 |
| 审计    | 审计详情     | `/admin/audit/{eventId}`    | 单条审计事件追踪与排障               |
| 运维    | 系统健康     | `/admin/health`             | 元数据库、MCP、数据源池、配置源健康状态     |

### 全局布局

- **左侧导航**：固定 220px 左右，按“工作台 / 身份与访问 / 配置与策略 / 审计 / 运维”分组。
- **顶部栏**：管理员账号、全局健康摘要、当前配置源、审计快速搜索、退出登录。
- **内容区**：顶部为页面标题和主要操作，下一行为筛选区，主体优先使用表格或详情分栏。
- **反馈组件**：加载使用 skeleton 行；危险操作使用确认弹窗；策略拒绝使用可复制的拒绝原因和审计编号。
- **标识展示**：`request_id`、`sql_hash`、`token_prefix` 使用等宽字体；状态使用稳定颜色，不依赖装饰图形。

## 页面清单

### 登录

| 项    | 内容                                     |
|------|----------------------------------------|
| 主任务  | 管理员完成登录，或在会话过期后重新进入后台。                 |
| 关键字段 | 用户名、密码、登录错误、会话过期提示、CSRF 隐藏字段。          |
| 主要交互 | 提交登录、显示/隐藏密码、回车提交、登录失败后保留用户名、退出后返回登录页。 |
| 空状态  | 首次部署未初始化管理员时，展示“管理员账号未配置”的阻断提示和配置入口说明。 |
| 错误状态 | 账号不存在、密码错误、账号停用、认证服务不可用、CSRF 校验失败。     |
| 加载状态 | 登录按钮进入禁用态并显示处理中，避免重复提交。                |

### 总览

| 项    | 内容                                                                                     |
|------|----------------------------------------------------------------------------------------|
| 主任务  | 用一屏判断 DBFlow 是否可用、是否存在高风险请求、是否有需要处理的异常。                                                |
| 关键字段 | MCP endpoint 状态、元数据库状态、配置源状态、项目数、环境数、用户数、有效 Token 数、近 24 小时 SQL 执行数、拒绝数、高风险数、失败数、待确认数。 |
| 主要交互 | 时间范围切换、按环境筛选、刷新、跳转审计列表、跳转系统健康、跳转危险策略。                                                  |
| 空状态  | 新系统暂无审计和授权时，展示初始化引导：创建用户、签发 Token、配置项目环境授权。                                            |
| 错误状态 | 统计查询失败、元数据库不可达、配置源不可达；保留可用模块并标记局部失败。                                                   |
| 加载状态 | 指标卡 skeleton、最近事件表格 skeleton、健康摘要占位。                                                   |

### 用户管理

| 项    | 内容                                                     |
|------|--------------------------------------------------------|
| 主任务  | 管理后台用户，控制谁可以申请 Token 和访问项目环境。                          |
| 关键字段 | 用户 ID、用户名、显示名、角色、状态、授权环境数、有效 Token 数、最近使用时间、创建时间、更新时间。 |
| 主要交互 | 创建用户、编辑用户、启用/停用用户、按用户名搜索、按状态筛选、跳转用户 Token、跳转用户授权。      |
| 空状态  | 暂无用户时展示“创建第一个管理员/用户”主操作。                               |
| 错误状态 | 用户名重复、当前用户不能停用自己、权限不足、保存失败。                            |
| 加载状态 | 用户表格 skeleton，行级操作按钮禁用。                                |

### Token 管理

| 项    | 内容                                                                 |
|------|--------------------------------------------------------------------|
| 主任务  | 为用户签发 MCP 访问 Token，查看使用状态并及时吊销风险 Token。                            |
| 关键字段 | Token ID、所属用户、Token 前缀、状态、生效时间、过期时间、最近使用时间、最近客户端、最近来源 IP、创建人、吊销时间。 |
| 主要交互 | 签发 Token、签发成功后单次显示明文并提供复制、吊销 Token、按用户/状态/过期时间筛选、查看关联审计。           |
| 空状态  | 用户暂无 Token 时展示“签发 Token”入口，并提示明文只显示一次。                             |
| 错误状态 | 用户不存在、用户已停用、过期时间非法、吊销失败、无权查看他人 Token。                              |
| 加载状态 | Token 表格 skeleton，签发弹窗提交按钮进入处理中。                                   |

### 项目环境授权

| 项    | 内容                                                   |
|------|------------------------------------------------------|
| 主任务  | 维护用户对指定 `project/env` 的访问权限，确保 SQL 执行前可以做授权判定。       |
| 关键字段 | 项目 Key、项目名、环境 Key、环境名、用户、授权状态、授权人、授权时间、最近访问时间、数据源状态。 |
| 主要交互 | 授权用户、撤销授权、按用户/项目/环境筛选、查看环境配置摘要、跳转授权相关审计。             |
| 空状态  | 当前项目环境暂无授权时，展示“添加授权”入口和授权生效范围说明。                     |
| 错误状态 | 项目不存在、环境不存在、重复授权、撤销后仍有执行中的确认请求、权限不足。                 |
| 加载状态 | 授权矩阵或授权表格 skeleton，保存时行级锁定。                          |

### 配置查看

| 项    | 内容                                                                         |
|------|----------------------------------------------------------------------------|
| 主任务  | 查看当前生效的项目、环境和数据源配置，辅助排查连接与策略问题。                                            |
| 关键字段 | 项目 Key、环境 Key、配置来源、数据源名称、数据库类型、主机、端口、库名、用户名、连接池大小、查询超时、最大返回行数、最近加载时间、配置版本。 |
| 主要交互 | 按项目/环境筛选、刷新配置、展开数据源详情、复制非敏感字段、跳转系统健康。                                      |
| 空状态  | 没有任何项目环境配置时，展示配置源位置和下一步配置提示。                                               |
| 错误状态 | 配置解析失败、配置源不可达、数据源缺少必要字段；不展示密码或完整 JDBC URL。                                 |
| 加载状态 | 配置树 skeleton，详情区域显示读取中。                                                    |

### 危险策略

| 项    | 内容                                                                     |
|------|------------------------------------------------------------------------|
| 主任务  | 查看高风险 SQL 的允许范围、确认要求和拒绝原因，帮助管理员解释策略命中。                                 |
| 关键字段 | 操作类型、风险等级、项目、环境、schema、table、是否允许生产环境、是否需要确认、确认 TTL、白名单来源、策略版本、最近更新时间。 |
| 主要交互 | 按操作类型/风险等级/项目/环境筛选、查看策略详情、查看匹配示例、跳转被拒绝审计。                              |
| 空状态  | 暂无高危白名单时展示默认策略：`DROP DATABASE`、`DROP TABLE` 拒绝，`TRUNCATE` 需要确认。        |
| 错误状态 | 策略文件解析失败、白名单冲突、策略版本缺失、配置源不可达。                                          |
| 加载状态 | 策略表格 skeleton，筛选条件保留。                                                  |

### 审计列表

| 项    | 内容                                                                             |
|------|--------------------------------------------------------------------------------|
| 主任务  | 管理员按条件追踪请求、策略决策、SQL 执行和异常事件。                                                   |
| 关键字段 | 时间、用户、Token 前缀、客户端、来源 IP、项目、环境、工具、操作、风险等级、决策、SQL Hash、结果摘要、耗时、审计事件 ID。         |
| 主要交互 | 按时间/用户/项目/环境/risk/decision/SQL hash/tool 筛选，分页，排序，复制 SQL Hash，打开审计详情，保留查询条件返回。 |
| 空状态  | 筛选条件下无结果时展示可清除的筛选项；全库暂无审计时提示系统尚未收到请求。                                          |
| 错误状态 | 查询超时、权限不足、排序字段不支持、过滤条件格式错误。                                                    |
| 加载状态 | 表格 skeleton，分页控件禁用但保留当前页码。                                                     |

### 审计详情

| 项    | 内容                                                                                                                                    |
|------|---------------------------------------------------------------------------------------------------------------------------------------|
| 主任务  | 还原一次请求的安全上下文、策略决策、执行结果和失败原因。                                                                                                          |
| 关键字段 | 审计事件 ID、请求时间、用户 ID、Token ID、客户端名称、客户端版本、User-Agent、来源 IP、项目、环境、工具、operation、risk、decision、SQL 文本、SQL Hash、结果摘要、影响行数、错误信息、确认 ID、事件时间线。 |
| 主要交互 | 返回列表并保留筛选条件、复制 `request_id`、复制 `sql_hash`、跳转用户、跳转项目环境授权、跳转危险策略、查看脱敏说明。                                                                |
| 空状态  | 审计事件不存在时展示“记录不存在或已归档”。                                                                                                                |
| 错误状态 | 无权查看详情、事件加载失败、关联用户或项目已删除。                                                                                                             |
| 加载状态 | 分区 skeleton：基础信息、SQL、策略、结果、时间线分别占位。                                                                                                   |

### 系统健康

| 项    | 内容                                                                                                |
|------|---------------------------------------------------------------------------------------------------|
| 主任务  | 判断 DBFlow 后台、MCP 网关、元数据库、配置源和目标数据源是否健康。                                                           |
| 关键字段 | 应用版本、启动时间、MCP endpoint、元数据库状态、Flyway 版本、配置源状态、最近配置刷新时间、数据源池状态、活跃连接、等待连接、最近连接错误、JVM 内存、线程、最近失败审计数。 |
| 主要交互 | 手动刷新、只看异常、按项目/环境筛选数据源池、跳转配置查看、跳转相关审计。                                                             |
| 空状态  | 暂无目标数据源时展示“仅基础服务健康”的说明。                                                                           |
| 错误状态 | 健康检查超时、元数据库不可达、配置源不可达、目标数据源连接失败。                                                                  |
| 加载状态 | 健康分组 skeleton，局部健康检查显示单项读取中。                                                                      |

## 跨页面状态规范

| 状态   | 设计要求                                                 |
|------|------------------------------------------------------|
| 加载   | 表格使用 5 到 8 行 skeleton，详情页按区域 skeleton，避免整页大 spinner。 |
| 空    | 展示当前筛选条件、原因和下一步操作；不使用插画占据主要空间。                       |
| 错误   | 展示错误类型、可复制错误码或审计编号、重试入口；安全拒绝必须说明 policy decision。    |
| 无权限  | 明确提示当前管理员权限不足，保留返回入口，不展示敏感字段。                        |
| 脱敏   | Token 明文、数据库密码、完整连接串、secret 永不在列表和详情页展示。             |
| 高危操作 | 使用二次确认弹窗，展示影响范围、风险等级、确认 TTL 和审计记录入口。                 |

## P09.2 HTML 原型承接要求

后续 `docs/prototypes/admin/index.html` 应按本 README 一比一落地以下内容：

- 固定后台 Shell：左侧导航、顶部状态栏、主内容区。
- 至少覆盖 10 个页面状态入口，页面名称与本文保持一致。
- 审计列表必须具备多条件筛选、分页、排序和详情跳转。
- Token 签发必须具备“一次性明文展示”弹窗。
- 项目环境授权必须具备授权/撤销交互。
- 危险策略必须展示策略拒绝原因和确认要求。
- 系统健康必须展示基础服务、配置源、元数据库、目标数据源池的分组状态。
